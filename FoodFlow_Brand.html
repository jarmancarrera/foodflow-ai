Why existing solutions fail
📞 Manual phone calls
Staff calls food banks at 5:55 PM. No answer. Food goes to dumpster.

This happens in 60% of attempted donations. Timing is the failure mode, not intent. FoodFlow starts the process 2 hours before closing — automatically.

🛍️ Too Good To Go
Consumer magic bags — wrong category entirely.

Requires retail packaging and consumer pickup. Doesn't serve 200 lb bulk surpluses at university dining halls or hotel kitchens.

⚖️ Liability fear (the hidden barrier)
Institutions cite legal risk as their #1 reason not to donate.

The Bill Emerson Good Samaritan Act protects them — but nobody verifies compliance automatically. FoodFlow does, on every rescue.

⚡ FastAPI Application main.py
6 routes: GET / dashboard · GET /api/stats · GET /api/surplus · POST /api/trigger/{id} → spawns BackgroundTask → GET /api/report. Uvicorn ASGI.
Layer 3 — Data
🗄️ SQLite Database database.py
Tables: food_inventory, food_banks, volunteers, pickups. Seed data: Cornell Statler Hall, RPCC, Ithaca Food Bank, 3 drivers. Helpers: get_stats(), get_surplus_items().
Layers 4 & 5 — AI Agent + Reporting
🤖 Claude Agent Loop agent.py
run_agent(trigger) → while loop → Anthropic API call with TOOL_SCHEMAS → dispatches TOOL_FUNCTIONS[block.name](**block.input) → appends tool_result → repeats until stop_reason == "end_turn". Model: claude-sonnet-4-6.
🔧 Tool Functions + Schemas tools.py
TOOL_FUNCTIONS dict maps 6 names → Python callables. TOOL_SCHEMAS list is the JSON schema array passed to Anthropic API — Claude reads this to understand each tool's input types, required fields, and purpose.
📄 ESG PDF Generator report.py
reportlab PDF: cover page, impact stats (lbs rescued, meals, CO₂), compliance audit log. Triggered via GET /api/report. Replaces $500/quarter manual reporting engagement.
🔄 One-click rescue flow
Browser → POST /trigger/statler → BackgroundTask → agent.py → Anthropic API ↔ tools.py ↔ database.py → pickup logged → dashboard refresh

The Core Code — Annotated
# agent.py — FoodFlow's orchestration brain def run_agent(trigger: dict) -> dict: messages = [{ "role": "user", "content": f"Rescue surplus at {trigger['location']}" }] # Keep calling Claude until it says "I'm done" while True: response = client.messages.create( model="claude-sonnet-4-6", tools=TOOL_SCHEMAS, # Claude reads these to know what's available messages=messages ) # Claude is done — return natural language summary if response.stop_reason == "end_turn": return {"summary": response.content[0].text} # Claude chose a tool — execute the Python function for block in response.content: if block.type == "tool_use": result = TOOL_FUNCTIONS[block.name](**block.input) # Send result back so Claude reasons on it messages.append({"role": "user", "content": [{ "type": "tool_result", "tool_use_id": block.id, "content": json.dumps(result) }]})
Step-by-Step in Plain English
1
We send Claude a task in plain language
"There's surplus food at Statler Hall. Please coordinate a rescue." No specific instructions on HOW — Claude decides.
2
Claude returns a tool_use block
Instead of answering in text, Claude says: "I want to call check_inventory with location_id='statler'." This is stop_reason = "tool_use".
3
We run the Python function
TOOL_FUNCTIONS["check_inventory"](**{"location_id":"statler"}) queries SQLite, returns JSON with surplus items and quantities.
4
We give Claude the data — loop repeats
We append the tool_result to the conversation. Claude reads it, reasons, picks next tool. Repeats 5 more times across all 6 tools.
5
Claude signals done — stop_reason = "end_turn"
After dispatch_pickup succeeds: "Dispatched Sarah Chen (0.8mi) for 45 lbs from Statler Hall. ETA 12 min to Ithaca Food Bank. Compliance verified ✓"
💡 Why not just a script?

Scripts need hard-coded if-else for every edge case. Claude handles: "Food bank A is full → try B → B only takes cold food → our food is hot → escalate." That flexibility requires LLM reasoning.
