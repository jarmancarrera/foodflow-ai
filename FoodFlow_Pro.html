Why Existing Solutions Fail
📞
Manual phone calls
Staff calls food banks at 5:55 PM. No answer. Food goes to dumpster. This happens in 60% of attempted donations — timing is the failure mode, not intent.
🛍️
Too Good To Go
Consumer-facing "magic bags." Requires retail packaging, consumer pickup, and staff time. Doesn't work for 200 lb hot-food surpluses at institutions.
🔗
Feeding America
Logistics network, not technology. Still requires human calls, scheduling, and coordination. No AI prediction or autonomous dispatch layer.
⚖️
Liability fear
Institutions cite legal risk as their #1 reason not to donate. The Bill Emerson Act protects them — but nobody verifies compliance automatically. FoodFlow does.

The Solution
Claude claude-sonnet-4-6 tool_use agent — 6 tools, zero human coordination
🌿 FoodFlow AI
FoodFlow connects to a restaurant's inventory system, detects surplus 2 hours before closing, finds the nearest food bank, locates a volunteer driver, verifies legal compliance, and dispatches the pickup — all in ~4 seconds, with zero staff involvement.

Layer 2 — API
⚡ FastAPI Application main.py
6 routes: GET / (dashboard), GET /api/stats, GET /api/surplus, POST /api/trigger/{id} → spawns BackgroundTask(run_agent), GET /api/pickups, GET /api/report. Uvicorn ASGI server.
Layer 3 — Data
🗄️ SQLite Database
database.py · foodflow.db
Tables: food_inventory, food_banks, volunteers, pickups. Seed data: Cornell Statler Hall, RPCC, Ithaca Food Bank, 3 volunteer drivers. get_stats() / get_surplus_items() helpers.
Layer 4 — AI Agent Core
🤖 Claude Agent Loop agent.py
run_agent(trigger) → while loop calling Anthropic API with TOOL_SCHEMAS → dispatches TOOL_FUNCTIONS[block.name](**block.input) → appends tool_result → repeats until stop_reason=="end_turn". Model: claude-sonnet-4-6, max_tokens: 4096.
🔧 Tool Functions + Schemas tools.py
TOOL_FUNCTIONS dict maps 6 tool names to Python callables. TOOL_SCHEMAS list is the JSON schema array passed to Anthropic API — defines input types, required fields, descriptions that Claude reads to understand each tool's capabilities.
Layer 5 — Reporting
📄 ESG PDF Generator
report.py
reportlab PDF: cover page, impact stats table (lbs rescued, meals, CO₂ avoided), compliance log, trend data. Triggered via GET /api/report — auto-downloads PDF. Replaces $500/quarter manual reporting.
🔄 Request Flow (one button click → rescue dispatched)
Browser → POST /api/trigger/statler → BackgroundTask → agent.py → Anthropic API ↔ tools.py ↔ database.py → dispatch logged → dashboard refresh

How the Agent Loop Works
agent.py line-by-line — explained for everyone
🌿 FoodFlow AI
The Core Code — Annotated
# agent.py — FoodFlow's orchestration brain def run_agent(trigger: dict) -> dict: messages = [{ "role": "user", "content": f"Rescue surplus at {trigger['location']}" }] # Loop until Claude says "I'm done" while True: response = client.messages.create( model="claude-sonnet-4-6", tools=TOOL_SCHEMAS, # Claude reads these to know what tools exist messages=messages ) # Claude is done — return its natural language summary if response.stop_reason == "end_turn": return {"status": "complete", "summary": response.content[0].text} # Claude wants to call a tool — execute it for block in response.content: if block.type == "tool_use": result = TOOL_FUNCTIONS[block.name](**block.input) # Send the result back so Claude can reason on it messages.append({"role": "user", "content": [{ "type": "tool_result", "tool_use_id": block.id, "content": json.dumps(result) }]})
Step-by-Step in Plain English
1
We send Claude a task
"There's surplus food at Statler Hall. Please coordinate a rescue." — natural language, no specific instructions on HOW.
2
Claude decides to call a tool
Instead of text, Claude returns: "I want to call check_inventory with location_id='statler'". This is the tool_use block.
3
We run the Python function
TOOL_FUNCTIONS["check_inventory"](**{"location_id": "statler"}) queries SQLite, returns JSON with surplus items.
4
We give Claude the data
We append tool_result to the conversation. Claude reads the JSON, reasons about it, and picks the next tool. Repeats 5 more times.
5
Claude returns stop_reason = "end_turn"
After dispatch_pickup succeeds, Claude writes: "Dispatched Sarah Chen (0.8mi) to pick up 45lbs from Statler Hall. ETA: 12 min to Ithaca Food Bank."
💡 Why This Beats Rule-Based Logic
A script needs hard-coded if-else for every edge case. Claude handles: "Food bank A is full → try B → B only takes cold food → our food is hot → escalate to human." That flexibility is impossible without LLM reasoning.
